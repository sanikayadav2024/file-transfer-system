# 🤝 Contributing to QuickShare

First of all, thank you for considering contributing to **QuickShare**! 🎉

Whether you're fixing bugs, improving documentation, adding new features, or suggesting ideas, every contribution is appreciated.

---

# 📌 Before You Start

Please make sure to:

- ⭐ Star this repository
- 🍴 Fork the repository
- 📖 Read the README carefully
- 🔍 Check existing issues before creating a new one

---

# 🚀 Getting Started

## 1. Fork the Repository

Click the **Fork** button at the top-right of this repository.

## 2. Clone Your Fork

```bash
git clone https://github.com/<your-username>/file-transfer-system.git
```

Replace `<your-username>` with your GitHub username.

---

## 3. Navigate to the Project

```bash
cd file-transfer-system
```

---

## 4. Create a New Branch

Create a branch for your changes.

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b feature/add-dark-mode
```

---

## 5. Make Your Changes

You can contribute by:

- Fixing bugs
- Improving UI
- Updating documentation
- Refactoring code
- Adding new features
- Optimizing performance

---

## 6. Commit Your Changes

Write clear and meaningful commit messages.

Example:

```bash
git add .

git commit -m "Add drag-and-drop support"
```

---

## 7. Push Your Branch

```bash
git push origin feature/your-feature-name
```

---

## 8. Open a Pull Request

Go to your fork on GitHub and click **New Pull Request**.

Please describe:

- What you changed
- Why you changed it
- Any screenshots (if UI changes)
- Related issue number

---

# 📝 Coding Guidelines

Please follow these guidelines:

- Write clean and readable code.
- Follow Java naming conventions.
- Keep methods small and focused.
- Remove unused imports.
- Add comments where necessary.
- Test your changes before submitting.

---

# 🐞 Reporting Bugs

When creating a bug report, include:

- Operating System
- Java Version
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)

---

# 💡 Feature Requests

Feature requests are welcome!

Please include:

- A clear description
- Why the feature is useful
- Possible implementation ideas (optional)
